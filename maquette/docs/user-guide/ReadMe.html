Xemelios v3.0.0 beta3

____________

Objet de Xemelios

Xemelios est un outil permettant l'exploitation de documents XML : recherche multicritère, tri de résultats, visualisation et export.
Cet outil est disponible sous licence GPL.

Cet Outil s'inscrit pleinement dans le cadre de la Charte Nationale de Dématérialisation signée en décembre 2004 par les acteurs de la chaine comptable & financières dans le secteur Public Local.

Sa première application concrète concerne la dématérialisation des états justificatifs de la Paye, états adressés par les ordonnateurs au comptable du Trésor dans le cadre du mandatement de la Paye.
Le schéma XML arrêté par tous les partenaires est annexé au présent outil.

L'outil peut être étendu par paramétrage à d'autres objets XML. Dans le futur, cet outil permettra notamment de lire et exploiter le compte de gestion sur chiffres dématérialisé par le Trésor Public.

Pour plus de précisions, http://www.colloc.minefi.gouv.fr/colo_struct_fina_loca/comp_loca/dema_chai_3.html

____________

Nouveautés

2.0.0
- Fonctionnement en base partagée. L'import ne se fait qu'une fois sur le poste principal.
- Visualisation des périodes importées (les critères année & mois sont prérenseignés avec les périodes importées)
- Sauvegardes des recherches : les recherches complexes peuvent être sauvagardêes en local du poste client
- Les recherches prédéfinies (comparaison entre mois calculs des entrants et sortants, ou variation des montants en % ou euros)
- L'export de toutes ou partie des lignes des états de paye 
- Suppression sélective des mois. Certaines périodes peuvent ainsi être supprimées de la base après import (suppression des données qui ne sont plus utilisées)
- Affichage de certaines colonnes supplémentaires dans les résultats. Une option "Afficher le champ" peut être activée aprés sélection des champs concernés (exemple "Heures Supplémentaires").

Prérequis Techniques

Xemelios nécessite l'installation du Java Runtime Environnement 5.0 (JRE 1.5) disponible sur le site java.sun.com
Cette installation est réalisée automatiquement dans le package complet.


Les Prérequis matériels sur le PC dépendent fortement de la volumétrie des données XML traitées et du type d'objet XML.

Pour l'état de Paye particulièrement exigeant étant donné le nombre de données à indexer, les prérequis estimés sont les suivants :


1) POSTE EN MODE AUTONOME OU SERVEUR

Jusqu'à 5000 Payes annuelles
OS : Windows 98/2000/XP
RAM  minimum : 64 Mo (W98) ou 128 Mo (2000/XP)
RAM conseillée : 128 Mo (W98) ou 256 Mo (2000/XP)
Espace disque temporaire : 1 Go
Espace disque permanent : 300 Mo


Jusqu'à 50 000 Payes annuelles
OS : 2000/XP
RAM  minimum : 256 Mo (2000/XP)
RAM conseillée : 512 Mo et > (2000/XP)
Espace disque temporaire : 5 Go
Espace disque permanent : 1 Go


Au dessus de 50 000 Payes annuelles
Nécessite la mise en place d'un serveur "Base de données"
Nous consulter : helios.demat@dgfip.finances.gouv.fr


2) POSTE EN MODE CLIENT

OS : Windows 98/2000/XP
RAM  minimum : 64 Mo (W98) ou 128 Mo (2000/XP)
RAM conseillée : 128 Mo (W98) ou 256 Mo (2000/XP)
Espace disque permanent : 50 Mo
